Lazy loading demo using IntersectionObserver

This page demonstrates lazy loading using the IntersectionObserver API. Images are not requested immediately. Instead, JavaScript loads an image only when it is about to enter the viewport. This approach gives predictable, browser controlled behavior and works consistently across modern browsers.

How this will work

The HTML renders image placeholders. The real URL is stored in a data attribute. When an image wrapper intersects the viewport, JavaScript swaps the placeholder with the real ImageKit URL.
